Memory

学生時代

　中学、高校では野球部に所属していました。光高校は甲子園出場経験があり、設備も整っていましたが、その分練習量や上下関係は厳しかったです。しかし、高校野球で厳しい環境に身を置けたことで、継続して努力することの重要性や努力することの楽しさを理解できました。

消防学校入校

　高校卒業後は地元の光消防に入りました。倍率が高く難しいかもと感じていましたが、部活引退後から毎日9時〜12時まで勉強すると決めて実行したこと、消防の業務について事前に勉強したこと、高校時代の頑張りが認められたことが功を奏し、採用していただけました。
　その後7ヶ月間、県内の採用者が集められ消防に必要な知識及び技術を学ぶ消防学校に入校しました。消防学校では、現場活動におけるストレス耐性を高めるため、肉体的及び精神的に高負荷な状態で座学や訓練を行いました。正直かなり過酷な環境でしたが、同期で支え合い一人も欠けることなく卒業しました。

中央消防署配属(1〜5年目)

　消防学校卒業後は光消防に戻り、中央消防署という出動メインの部署に配属されました。当時の上司が「現場での判断力を身につけるために自分で調べ判断する力の育成」に重きを置かれていたため、すべて丁寧に教えてもらえるわけではないことに学生と社会人の違いを痛感しました。最初は困惑しましたが今ではこの力にとても助けられているため、この力の重要性が身に染みています。

[業務内容]

  • 消防隊、救急隊での出動
  • 現場活動のための訓練
  • 予防業務

　出動及び訓練は消防のメイン業務です。予防業務は、消防法令や火災予防条例を根拠に、事業所への査察や届出の処理を行います。届出の処理は主にExcelや専用システムでデータを入力します。Excelに表や関数を挿入し、全体の作業効率を上げたりしていました。
　その他にも、私が主体となって月に1回程度ですが部署内で勉強会を行っていました。これは重要な知識の再確認や国が新規発出した通知の確認などを主に行い、法令や条例、過去の事例などを参考に資料作成を行いました。
　2年目からは後輩も入ってきたので、教育用に業務内容をまとめた資料作成を行いました。これは主にWordを使用し校閲機能を活用しました。この作業を行なっている時にもっと自由に資料を作れたらと、ほんの少しですがプログラミングに興味を持ち始めました。
　実家が自営業なので、休日は実家の手伝いに出ていました。主に建築関係の仕事をしていたのですが、消防での資料作成や実家の手伝いの経験から「自分は物作りが好きなんだな」と実感し始めました。

通信指令室配属(6年目)

　6年目は通信指令室という通報対応メインの部署に配属されました。現場出動がなくなった代わりに少人数部署のため、事務作業の量が格段に増加しました。

[業務内容]

  • 119番通報対応
  • 出動隊の指揮
  • システムのデータ入力
  • システムの保守管理及び予算計上

　通報対応は、電話越しに現場状況を詳細に把握し出動隊に伝達する必要があるため、高度な情報収集能力が求められます。また、緊急通報なので通報者は相当焦っている状況が多く、対人能力も鍛えられました。通報切断後も出動隊の管理、必要に応じた関係機関への連絡などを行うため、全体を俯瞰的に管理する力が身につきました。
　システム関係の業務は、専用システムをユーザーとして活用していました。これは毎月データ入力が必要となりPCスキルが身につきました。また、保守管理のために導入業者と密に連携を取りつつ、予算と照らし合わせて調整を行いました。このシステムを使用している時に明確にプログラミングについて興味を持ち、勉強を始めました。簡単にですがHTML/CSを独学したおかげで、システムについて他の職員より理解度が深まり、これまであまり活用されていなかった部分の使用を提案することができました。

PR

転職のきっかけ

　私が消防からの転職を考えたきっかけは、プログラミングスキルを活かした仕事をしたいと考えたからです。ユーザーとしてシステムを使っていくうちに、もっと使い勝手を良くすることはできないのかという疑問が湧きました。そしてHTMLやCSSを学んでいくうちに、使う側ではなく作る側になれたらという気持ちが強くなっていきました。実際に勉強してみて難しいことも多いと感じていますが、それ以上に自分の手でコードを書き、物を作ることにやりがいを感じています。

消防で得たスキル

  • 根性
  • 根拠を意識した業務遂行
  • 対人能力
  • 自走能力
  • 学んだことを引き出す能力
  • 連携力
  • ユーザー目線での考え方

　Memory部分と被る部分が多々あります、申し訳ありません。
　高校時代から現在まで厳しい環境に身を置くことが多かったため、根性には自信があります。ただ闇雲に頑張るだけでなく、業務の目的や根拠を意識してより効率的な遂行方法を模索しながら業務にあたることができます。
　対人能力や自走能力は、中央消防署や通信指令室勤務の部分でもアピールさせていただきましたが、現場活動、通報対応、業者との連携の際に鍛えられたと感じています。また、1年目の上司の教えを今でも意識しており、自身で調べ実行することを癖付けています。
　私は決して物覚えが良いわけではありません。学んだことを忘れることも多々あります。しかし、それを自覚しているため、忘れてもすぐに引き出せるように資料化して保存する、インデックスをつけておく、メモしておくなどの対策を取ることができます。
　消防はチームで動くため、周囲との連携は非常に重要になります。特に情報共有ですが、自分の持っている業務の進行状況や問題点などの報告、他の人の業務の進行状況の確認などを自発的に行うことができます。
　その他に自身がユーザーとなってシステムを使用していた経験を、ユーザー目線を重視した開発に活かせたらと思っています。

自身の特徴

　消防を選んだ理由もですが、人の役に立てる仕事をしたいと考えています。職種は全く変わりますが、エンジニアとしてユーザーが快適に使用できるものを開発、運用し喜んでいただたらと考えています。
　また、行動力がとても重要だと思っているので、まず動くことを意識しています。目的のための努力を楽しめるタイプなので、困難な壁に当たった際も投げ出さずにやりきります。

目標

　プライベートな目標にはなりますが、親への恩返しとして、ある程度知識や技術が身についたタイミングで、実家の自営業のHPや簡易的な事務処理ツールなどを作成できたらと考えています。全て1人でやる必要があり、自身の勉強にもなるので、職場への還元にも繋げれると思っています。
